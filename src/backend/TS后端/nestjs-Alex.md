## 安装

1. 安装命令: `npm install -g @nestjs/cli`
2. 创建项目: `nest new project-name`可以简写为`nest n project-name`
   - 跳过安装git: `nest new project-name -g`
   - 跳过安装依赖: `nest new project-name -s`
3. 启动项目: `nest start`

## IoC和DI

- `IoC` (Inversion of Control，控制反转)：对象控制权交给框架, 即对象的创建、依赖管理、生命周期控制，不再由业务代码自己负责，而是交给框架或容器负责。
- `DI` (Dependency Injection，依赖注入)：框架把依赖自动注入给你
- `AOP`：把日志、权限、异常等通用逻辑从业务代码中抽离出来

在 NestJS 中，`@Injectable()` + `constructor 注入` 就是 DI，`Guard` / `Interceptor` / `Pipe` / `Filter` 就是典型 AOP。

## \*passthrough

当使用`@Res`/`@Next`时, nestjs自动处理返回值的方式会被禁用

如果想同时使用`@Res`/`@Next`和nestjs内置的自动`json`处理功能，可以使用`@Res({passthrough: true})`

## DTO

一般把Controller接受到的数据类型(参数)叫做`DTO`(Data Transfor Object). nestjs推荐使用`class`形式来创建DTO:

```ts
// create-cat.dto.ts
export class CreateCatDto {
  name: string;
  age: number;
  breed: string;
}
```

::: tip PartialType

```ts
import { PartialType } from '@nestjs/mapped-types'; // 需要单独安装`@nestjs/mapped-types`
export class UpdateCatDto extends PartialType(CreateCatDto) {
  // CreateCatDto所有字段会继承过来，但是是可选的，可以部分更新
}
```

未来想要改动只需要改动`CreateCatDto`
:::

## 校验和Pipe

1. 安装依赖

```sh
pnpm add class-validator class-transformer # 校验 类型转换
```

2. 配置

```ts
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalPipes(new ValidationPipe()); // 全局注册校验
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

3. 可以在`DTO`中添加`@IsString()`等校验规则
4. 可以在`Controller`中添加`ParseIntPipe`: `findOne(@Param('id', ParseIntPipe) id: string)`来进行数据转换
   - 也可以在`Controller`中添加`@UsePipes(new ParseIntPipe({transform: true}))`来进行数据转换
   - 在全局注册校验位置添加

```ts
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalPipes(
    new ValidationPipe({
      transform: true, // 添加转换
      whitelist: true, // 未经验证的字段会被过滤掉
      forbidNonWhitelisted: true, // 未经验证字段会报错
    }),
  );
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

添加后, 所有类型都能根据`Controller`中定义的参数类型自动转换成对应的类型

## API文档

1. 安装:

```sh
pnpm add @nestjs/swagger
```

2. 在`main.ts`中配置:

```ts
// main.ts
const config = new DocumentBuilder()
  .setTitle('Cats example') // 文档标题
  .setDescription('The cats API description') // 文档描述
  .setVersion('1.0') // 文档版本
  .addTag('cats') // 文档标签
  .build();
const documentFactory = () => SwaggerModule.createDocument(app, config);
SwaggerModule.setup('api', app, documentFactory);
```

直接去访问`http://localhost:3000/api`就可以访问文档

可以添加不同的装饰器来描述对应的接口, 也可以配置自动添加文档:

```json
// nest-cli.json
{
  "$schema": "https://json.schemastore.org/nest-cli",
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "deleteOutDir": true,
    "plugins": ["@nestjs/swagger"] // 添加插件
  }
}
```

如果发现在使用`PartialType`时在swagger中无法显示, 将`PartialType`的导入改为`@nestjs/swagger`

生成可以导入的json文件

```ts
// main.ts
const documentFactory = () => SwaggerModule.createDocument(app, config);
SwaggerModule.setup('api', app, documentFactory, {
  jsonDocumentUrl: 'swagger/json', // 生成可供导入的swagger json文件
});
```

## SWC

使用SWC加速构建

1. 安装

```sh
pnpm add -D @swc/cli @swc/core # 安装
```

2. 配置

```json
{
  "$schema": "https://json.schemastore.org/nest-cli",
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "deleteOutDir": true,
    "builder": "swc" // 配置SWC
  }
}
```

但是使用swc会导致之前配置的swagger无法使用, 可以选择在开发阶段使用默认的构建方式, 在生产阶段使用swc

::: tip 解决不兼容问题

```json
{
  "$schema": "https://json.schemastore.org/nest-cli",
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "deleteOutDir": true,
    "plugins": ["@nestjs/swagger"],
    "builder": "swc",
    "typeCheck": true // 配置类型检查
  }
}
```

:::

构建后会在src目录下生成`metadata.ts`, 该文件需要手动导入

```ts
// main.ts
import metadata from './metadata'; // <-- file auto-generated by the "PluginMetadataGenerator"

await SwaggerModule.loadPluginMetadata(metadata); // <-- here
const document = SwaggerModule.createDocument(app, config);
```
