3. 项目结构优化 ⭐⭐⭐ ​

当前问题：

  • 部署脚本 index.js 散落在根目录
  • stats.html 等分析文件混在根目录

⚠️ 注意：本项目的 docs/ 是构建输出目录，不是源码目录。 源码内容位于根目录的 src/，VitePress 通过 outDir: "docs" 把产物输出到 docs/ 供 GitHub Pages 发布。这与 VitePress 官方「源码放 docs/、产物输出到 docs/.vitepress/dist」的默认布局相反，重构时切勿把源码移进 docs/，否则会与 outDir 配置冲突并污染发布目录。

可选的轻量调整（不影响部署）：

项目根目录/
├── .vitepress/          # VitePress 配置
│   ├── config.mts
│   └── cache/
├── src/                 # 📝 文档源码（实际内容在这里）
│   ├── frontend/
│   ├── backend/
│   ├── web/
│   └── other/
├── public/              # 静态资源
├── docs/                # ⚙️ 构建输出（GitHub Pages 发布目录，需提交）
├── scripts/             # 🆕 可选：把 index.js 移到这里改名 deploy.js
│   └── deploy.js
├── .gitignore
├── package.json
├── index.md             # 首页
├── 简介.md
└── 主要内容概述.md

9. CI/CD 优化 ⭐⭐⭐ ​

⚠️ 不要照抄常见的 peaceiris/actions-gh-pages 示例！ 那种方案会把产物推到独立的 gh-pages 分支，而你现在是「main 分支 + /docs 文件夹」发布。两套机制混用会互相打架。要上 CI/CD，必须先二选一，不能直接套用。

两条可行路线（任选其一）：

路线 A：保持现有「main 分支 /docs 文件夹」模式，仅自动化构建提交

yaml
name: Build and Deploy Docs

on:
  push:
    branches: [main]
    paths-ignore:
      - "docs/**" # 避免提交产物后再次触发，造成死循环

permissions:
  contents: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - name: Commit built docs
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add docs
          git diff --staged --quiet || git commit -m "chore: rebuild docs [skip ci]"
          git push

采用路线 A 后，本地就不必再跑 npm run deploy，但要注意循环触发问题（已用 paths-ignore 规避）。

路线 B：改用 GitHub 官方 Pages Actions（推荐，产物不再入库）

切换为「GitHub Actions」发布源（仓库 Settings → Pages → Source 改为 GitHub Actions），此时 docs/ 可以不再提交：

yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs
      - id: deployment
        uses: actions/deploy-pages@v4
